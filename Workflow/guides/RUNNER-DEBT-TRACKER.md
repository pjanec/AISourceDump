- **DEBT-011: `UnsafeLayout<T>` / `MultiInstanceLayout<T>` 32-bit `EntityId` Network Discrepancy (CRITICAL)**
  - **Location:** `UnsafeLayout<T>.cs`, `MultiInstanceLayout<T>.cs`
  - **Issue:** The DDS BDC SSTD descriptor standard defines `int EntityId` (4 bytes). However, FDP's unsafe blitting classes (`UnsafeLayout` and `MultiInstanceLayout`) hardcoded a structural integrity check that required `typeof(long)` or `typeof(ulong)` (8 bytes), violently crashing translators at startup.
  - **Resolution Strategy**: Check for `IsEntityId32Bit`. Accept `typeof(int)`. Mask the 4 byte values and natively cast up to `long` before invoking lookup against the rigorous 64-bit `NetworkEntityMap`. 
  - **Status:** Resolved (Runner Batch 4). AutoCycloneTranslator<EntityMaster> restored.
